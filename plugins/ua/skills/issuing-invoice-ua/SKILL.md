# issuing-invoice-ua

Скіл — практичний посібник із виставлення різних типів первинних документів для ФОП в Україні.

## Ключові джерела

- **ЗУ «Про бухгалтерський облік та фінансову звітність в Україні»** (`996-14`), ст. 9 — обов'язкові реквізити первинного документа.
- **ПКУ** (`2755-17`):
  - Ст. 201 — податкова накладна.
  - Ст. 201.10 — строки реєстрації в ЄРПН.
  - Ст. 120¹ — штрафи за несвоєчасну реєстрацію.
  - Ст. 192 — коригуючі розрахунки.
  - Ст. 186.3 — місце надання послуг (для визначення, чи в полі ПДВ).
- **Наказ МФУ № 1307 від 31.12.2015** — форма ПН і РК.
- **Наказ МФУ № 13 від 21.01.2016** — форма чеку РРО/ПРРО.

## Типи документів — матриця вибору

| Сценарій | Первинний документ | Додатково |
|---|---|---|
| ФОП (без ПДВ) → ТОВ (B2B) | Акт наданих послуг | Рахунок на оплату (опц.) |
| ФОП (без ПДВ) → фізособа | Акт + чек ПРРО (якщо обов'язковий) | |
| ФОП (без ПДВ) → закордон (B2B) | English Invoice | Договір + CMR (для товарів) |
| ФОП (з ПДВ) → будь-кому | Акт + ПН (ЄРПН) | Рахунок на оплату (опц.) |
| Товари (купівля-продаж) | Видаткова накладна | + ПН якщо платник ПДВ |
| Експорт товарів | Видаткова + митна декларація | 0% ПДВ (за документами) |

## Обов'язкові реквізити (ст. 9 ЗУ № 996-14)

Будь-який первинний документ має містити:

1. **Назву документа** (акт, накладна, рахунок).
2. **Дату складання**.
3. **Назву підприємства/ФОП** (обох сторін).
4. **Зміст і обсяг операції**, одиницю виміру.
5. **Посади осіб**, відповідальних за здійснення операції.
6. **Підписи** (особистий — для паперу; КЕП — для електронного).

Додатково корисно:
- **Номер документа**.
- **Реквізити сторін** (РНОКПП/ЄДРПОУ, адреси).
- **Договір, на підставі якого виконується операція**.
- **Реквізити для оплати** (IBAN, банк).

## Нумерація

- **Унікальна** в межах ФОП.
- **Безперервна** рекомендується (розриви → питання при перевірці).
- Формат — вільний:
  - `АКТ-2026-042`.
  - `ИНВ-2026-01-003` (рік-місяць-номер).
  - `INV-2026-123`.

## Шаблон 4: Податкова накладна (для ФОП-платника ПДВ)

Структура затверджена Наказом МФУ № 1307 (обов'язкова форма).

**Ключові реквізити:**

- Порядковий номер / номер філії.
- Дата складання.
- Постачальник: ІПН, назва, адреса.
- Покупець: ІПН (або «не платник ПДВ»), назва, адреса.
- Підстава (договір, акт).
- Номенклатура товарів/послуг з ПКВД/УКТЗЕД.
- Одиниця виміру, кількість, ціна без ПДВ, сума ПДВ (20%), сума з ПДВ.
- Загальна сума.

**Реєстрація в ЄРПН:**
- Підписати КЕП.
- Надіслати через M.E.Doc / Fredo / Sonata / кабінет.
- Термін — 15 к.д. (або до 10-го наступного місяця для деяких).

## Сценарії

### Сценарій 1: Перший інвойс США-клієнту

**Запит:** «Перший клієнт у США, Acme Corp, заплатить $9 500 за квітень».

**Алгоритм:**
1. Перевірити, що ФОП гр. 3 без ПДВ (стандарт для IT).
2. Використати шаблон «English Invoice».
3. Заповнити:
   - INV-2026-042 (наступний номер).
   - Issue date / Due date.
   - Seller (свої дані).
   - Bill To (дані клієнта з W-9 або договору).
   - Services (опис + години + ставка + сума).
   - Payment: IBAN mono business + SWIFT.
4. Згенерувати PDF (Fakturownia, Invoicely, Wave, Zervant; або самописний Python/LaTeX).
5. Надіслати клієнту e-mail.
6. Очікувати оплати → запис у реєстр.

### Сценарій 2: Українська юрособа хоче «рахунок-фактуру»

**Запит:** «ТОВ-клієнт надіслав email: "треба рахунок-фактуру на 50 000 грн"».

**Відповідь:**
- «Рахунок-фактура» — традиційна назва для **рахунку на оплату** + акта. Для ФОП без ПДВ:
  - **Рахунок на оплату** (шаблон 3) — для виставлення суми.
  - **Акт наданих послуг** (шаблон 1) — після надання / оплати.
- Попередити клієнта: «Я ФОП 3 групи без ПДВ, тому ПДВ не нараховую. Надсилаю Рахунок на оплату + після надання послуг підпишемо Акт».

### Сценарій 3: Валюта прийшла, клієнт запитав документ

**Запит:** «Google перерахував $10 000 на mono business. Хочуть документ для своєї бухгалтерії».

**Відповідь:**
- Виставити **English Invoice** (якщо ще не виставляли).
- Після зарахування — можна опційно виставити **Receipt / Statement** (підтвердження оплати).

### Сценарій 4: Помилка в ПН після реєстрації

**Запит:** «ПН зареєстрована з помилкою в ціні. Що робити?»

**Відповідь:**
1. Скласти **Коригуючий розрахунок (РК)** на суму різниці.
2. Дата РК — поточна (день виявлення помилки).
3. Зареєструвати РК в ЄРПН (15 к.д.).
4. В декларації ПДВ за відповідний період внести коригування.

### Сценарій 5: Повернення клієнту

**Запит:** «Клієнт повернув товар на 20 000 грн. Що з ПН?»

**Відповідь:**
1. Скласти **РК на зменшення** ПН.
2. Сумма зменшення = 20 000 грн ПЗ.
3. Реєстрація в ЄРПН.
4. Якщо клієнт неплатник ПДВ → тільки РК (він не має ПК до зменшення).
5. Якщо клієнт платник → його обов'язок — **зареєструвати отриманий РК** для зменшення власного ПК.

## Безплатні PDF-генератори

- **Fakturownia.pl** (підтримує UA) — безкоштовно до 3 faktur/міс.
- **Invoicely** (EN) — безкоштовно.
- **Wave** (EN) — повний безкоштовний.
- **Zervant** (EN) — безкоштовно до 5 клієнтів.
- **Invoice Generator (invoice-generator.com)** — без реєстрації, PDF одразу.

## Автоматичне виставлення з банку

### mono business API

- `https://api.monobank.ua/` (публічне API для клієнтів).
- Webhooks — сповіщення про надходження.
- Можна автоматично:
  - Згенерувати Invoice при відкритті договору.
  - Позначити як «оплачений» при надходженні.

### MyDocs.in.ua

- Інтеграція з mono, privat24, sense.
- Автоматичний імпорт виписок → парсинг платежів.

## Типові помилки

| Помилка | Наслідок | Правильно |
|---|---|---|
| Виставляє «Рахунок-фактуру» для послуг без ПДВ | Неправильна назва | «Рахунок на оплату» + «Акт» |
| Забуває зазначити «не є платником ПДВ» | Клієнт може помилково очікувати ПН | Завжди чітко в документі |
| Номер англомовного інвойсу українською («Інвойс №1») | Плутає іноземних клієнтів | «Invoice No. INV-2026-001» |
| Ставить UAH в валютному Invoice | Клієнт не може перерахувати | Валюта = валюта договору (USD/EUR) |
| Пропускає реквізити SWIFT | SWIFT-платіж неможливий | Повні реквізити банку + intermediary |
| Не реєструє ПН у 15 днів | Штраф 10-100% | Реєстрація в термін |
| Ставить дату акту раніше, ніж була послуга | Формально порушення | Акт = дата факту надання |

## Обмеження

- Форма ПН (Наказ № 1307) — **не підлягає відхиленням**. Використовувати офіційне ПЗ (M.E.Doc і подібні).
- Для товарів — додатково потрібні ТТН, видаткові накладні, сертифікати відповідності.
- Для міжнародного експорту товарів — митні декларації (поза темою скіла).
- Для ФОП з РРО — чек замість частини первинних документів при B2C.
